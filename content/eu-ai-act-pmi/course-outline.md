# EU AI Act: essentials e azioni pratiche per PMI

## Informazioni Generali del Corso

**Titolo completo:** EU AI Act: essentials e azioni pratiche per PMI
**Sottotitolo:** Guida rapida alla conformità per manager non tecnici
**Livello:** Intermedio (nessun prerequisito tecnico richiesto)
**Formato:** Micro-learning modulare
**Durata totale:** 90-120 minuti
**Target audience:** Manager non tecnici, Project Manager, Marketing Manager, HR Manager, Legal/Compliance Officer, IT Lead in PMI italiane
**Lingua:** Italiano
**Standard tecnico:** SCORM 1.2

### Descrizione del corso

Questo corso fornisce alle PMI italiane una guida pratica e operativa per comprendere l'EU AI Act e adottare azioni concrete di conformità nei prossimi 6-12 mesi. Attraverso lezioni brevi, scenari interattivi e strumenti scaricabili, i partecipanti apprenderanno a classificare i propri sistemi AI, implementare controlli di governance minimi e pianificare un percorso di adeguamento normativo.

Il corso utilizza un approccio micro-learning con lezioni di 5-8 minuti, esempi italiani ed europei, e zero legalese. Ogni modulo combina teoria essenziale con esercizi pratici immediatamente applicabili in azienda.

### Prerequisiti

- Nessun prerequisito tecnico o di sviluppo software
- Familiarità base con strumenti di AI generativa (es. ChatGPT, Copilot)
- Esperienza professionale in una delle funzioni target (management, PM, marketing, HR, legal, IT)

---

## Obiettivi di Apprendimento SMART del Corso

Al termine del corso, i partecipanti saranno in grado di:

1. **Distinguere e classificare** le categorie AI Act (sistemi proibiti, ad alto rischio, GPAI) e identificare almeno 1 implicazione diretta per la propria funzione aziendale entro 5 minuti dall'analisi di un caso d'uso. *(Modulo 1)*

2. **Mappare** almeno un caso d'uso AI della propria azienda a una specifica categoria AI Act e identificare 3 requisiti minimi di conformità applicabili. *(Modulo 2)*

3. **Implementare** una checklist completa di AI governance coprendo policy, dataset, valutazioni del rischio, log, e human oversight, raggiungendo un punteggio minimo del 70% nel quiz sommativo. *(Modulo 3)*

4. **Pianificare** 5 azioni concrete per i prossimi 90 giorni, assegnando ruoli, definendo policy, valutando vendor e organizzando comunicazione interna, producendo un mini-piano operativo di 1 pagina. *(Modulo 4)*

## Struttura del Corso (4 Moduli)

### Modulo 1: Panoramica rapida e timeline AI Act

**Durata stimata:** 20-25 minuti

**Obiettivi specifici del modulo:**
- Spiegare cos'è l'EU AI Act e perché è rilevante per le PMI italiane
- Identificare le 4 categorie principali di sistemi AI secondo la normativa
- Riconoscere le date chiave della timeline di applicazione (2025-2027)
- Distinguere tra obblighi immediati e obblighi futuri per le PMI

**Argomenti/Topics:**

**1.1 Introduzione all'EU AI Act**
- Cos'è l'AI Act e perché nasce (rischi vs innovazione)
- Ambito di applicazione: chi deve preoccuparsi (fornitori, deployer, importatori)
- Differenza tra fornitori di AI e utilizzatori (deployer) - focus su PMI come deployer
- Esempi concreti: quando una PMI "usa AI" vs "sviluppa AI"

**1.2 Le 4 categorie di sistemi AI**
- **Sistemi proibiti:** manipolazione comportamento, social scoring, riconoscimento biometrico real-time (eccezioni law enforcement)
- **Sistemi ad alto rischio:** recruitment, credit scoring, infrastrutture critiche, gestione lavoratori, accesso servizi essenziali
- **General Purpose AI (GPAI):** modelli fondazionali come GPT, Claude, Gemini - obblighi per provider
- **Sistemi a rischio limitato/minimo:** chatbot, tool generativi semplici - obblighi di trasparenza
- Esempi italiani per ciascuna categoria

**1.3 Timeline cruciale 2025-2027**
- **2 febbraio 2025:** divieti sui sistemi proibiti (già in vigore)
- **2 agosto 2025:** obblighi per GPAI provider
- **2 agosto 2026:** obblighi per sistemi ad alto rischio (la data più rilevante per PMI)
- **2 agosto 2027:** obblighi completi per tutti
- Infografica interattiva: cosa devo fare e quando

**1.4 Implicazioni per funzione aziendale**
- Marketing: trasparenza contenuti AI-generated, chatbot
- HR: recruitment automatizzato, valutazione performance
- Customer Service: chatbot, automazione supporto
- IT/Data: gestione dataset, documentazione sistemi
- Legal/Compliance: policy, registri, DPIA
- Mini-attività: "Nella mia funzione, l'AI Act impatta su..."

**Attività di apprendimento:**

**Lezione interattiva con ramificazioni leggere (12-15 min):**
- Pagine con domande in-line: "La tua azienda usa AI per...? [Marketing/HR/Customer Service/Altro]"
- Contenuto si adatta mostrando esempi specifici per la funzione selezionata
- Infografica timeline cliccabile con tooltip esplicativi
- Video breve (6 min): "AI Act in 6 minuti: cosa cambia per le PMI"

**Quiz formativo (6 domande, 8-10 min):**
- Feedback immediato dopo ogni risposta
- Domande tipo:
  1. Quale sistema è PROIBITO dall'AI Act? [Multiple choice]
  2. Una PMI che usa ChatGPT per scrivere email è... [Fornitore/Deployer/Nessuno dei due]
  3. Entro quale data le PMI devono conformarsi per sistemi ad alto rischio? [Data]
  4. Quale funzione è considerata ad alto rischio? [Recruitment/Email marketing/Chatbot semplice]
  5. Un chatbot sul sito deve... [Scenario-based]
  6. Vero/Falso: Le PMI che usano solo ChatGPT non hanno obblighi [V/F con spiegazione]

**Tipo di valutazione:** Formativa (non bloccante, traccia tentativi)

**Note per il content creator:**
- **Tono:** Pratico, rassicurante, evita panico ma sottolinea urgenza
- **Esempi:** Usa casi italiani/europei: azienda manifatturiera che usa AI per controllo qualità, e-commerce con chatbot, studio professionale con AI per analisi contratti
- **Interattività:** Domande in-line non devono rallentare, massimo 2-3 scelte
- **Linguaggio:** Zero termini legali senza spiegazione; glossario pop-up per "deployer", "GPAI", "alto rischio"
- **Visuals:** Timeline colorata (rosso=proibito, arancione=alto rischio, giallo=GPAI, verde=basso rischio)

### Modulo 2: Mappa dei casi d'uso e rischio

**Durata stimata:** 25-30 minuti

**Obiettivi specifici del modulo:**
- Classificare casi d'uso AI aziendali nelle categorie AI Act
- Identificare requisiti minimi di conformità per ciascuna categoria
- Applicare il framework di classificazione a un caso reale della propria azienda
- Riconoscere red flag che indicano alto rischio

**Argomenti/Topics:**

**2.1 Framework di classificazione AI Act**
- Decision tree: domande chiave per classificare un sistema AI
  - Il sistema influisce su diritti fondamentali? (lavoro, credito, accesso servizi)
  - Il sistema prende decisioni automatizzate senza supervisione umana?
  - Il sistema tratta dati sensibili/biometrici?
  - Il sistema è trasparente verso l'utente finale?
- Flowchart interattivo: da caso d'uso a categoria

**2.2 Scenario 1 - Marketing: AI-generated content e chatbot**
- Caso: PMI e-commerce usa ChatGPT per product description e chatbot per assistenza clienti
- Classificazione: Rischio limitato (trasparenza richiesta)
- Requisiti: Informare utenti che interagiscono con AI, watermark contenuti AI-generated
- Red flag: se chatbot raccoglie dati sensibili o influenza decisioni di acquisto in modo ingannevole

**2.3 Scenario 2 - HR: Recruitment automatizzato**
- Caso: PMI usa software AI per screening CV e video-interviste
- Classificazione: Alto rischio (impatta lavoro e diritti fondamentali)
- Requisiti: Valutazione conformità, documentazione, human oversight, dataset quality, trasparenza verso candidati
- Red flag: bias nel training dataset, assenza supervisione umana, decisione completamente automatizzata

**2.4 Scenario 3 - Customer Service: Scoring e segmentazione**
- Caso: PMI usa AI per credit scoring o valutazione affidabilità cliente
- Classificazione: Alto rischio (impatta accesso a servizi finanziari)
- Requisiti: Registro AI, DPIA, spiegabilità decisioni, diritto a contestazione
- Red flag: opacità algoritmo, impossibilità spiegare decisione negativa

**2.5 Scenario 4 - Analisi dati: Predizioni interne**
- Caso: PMI usa AI per previsioni vendite, ottimizzazione inventario, analisi KPI
- Classificazione: Rischio minimo/limitato (decisioni interne, no impatto diritti terzi)
- Requisiti: Minimi (documentazione interna consigliata)
- Red flag: se usato poi per decisioni HR o verso clienti

**2.6 Scenario 5 - Prodotto: AI incorporata in prodotto venduto**
- Caso: PMI sviluppa software con componente AI venduto a clienti
- Classificazione: Dipende dall'uso finale (potenzialmente fornitore di sistema ad alto rischio)
- Requisiti: Valutazione caso per caso, documentazione tecnica, marcatura CE
- Red flag: se prodotto usato in contesti regolati (medicale, trasporti, finanza)

**Attività di apprendimento:**

**Scenario a bivi (15-18 min):**
- Presentazione dei 5 casi d'uso tipici
- Per ciascun caso, partecipante deve:
  1. Selezionare categoria AI Act [Drag & drop o multiple choice]
  2. Identificare 2-3 requisiti applicabili [Checklist]
  3. Ricevere feedback immediato con spiegazione
- Sistema ramificato: risposta errata → breve micro-lezione → riprova

**Esercizio pratico - Scheda di classificazione (8-10 min):**
- Partecipante compila scheda per un caso d'uso reale della propria azienda
- Template scaricabile (PDF compilabile o Excel):
  - Descrizione caso d'uso (max 100 parole)
  - Funzione aziendale coinvolta
  - Decision tree guidato (5-6 domande yes/no)
  - Output: categoria AI Act + lista requisiti minimi
- Opzionale: upload scheda compilata per feedback (se corso B2B con tutorship)

**Tipo di valutazione:** Formativa (scenario a bivi traccia risposte corrette; scheda è esercizio non valutato)

**Note per il content creator:**
- **Tono:** Analitico ma accessibile, usa "tu" e domande dirette ("Nella tua azienda, usi AI per...?")
- **Esempi:** Molto concreti e settoriali (manifattura, retail, servizi professionali, tech)
- **Interattività:** Scenario a bivi deve dare feedback costruttivo, non punitivo ("Non proprio. Ecco perché questo è alto rischio...")
- **Linguaggio:** Spiega "DPIA", "human oversight", "bias" in tooltip o glossario laterale
- **Visuals:** Flowchart decisionale chiaro, icone per categorie rischio, colori consistenti con Modulo 1
- **Scheda classificazione:** Template professionale, brandizzabile, utile anche post-corso

### Modulo 3: Governance e controlli minimi

**Durata stimata:** 30-35 minuti

**Obiettivi specifici del modulo:**
- Implementare una AI policy aziendale conforme all'AI Act
- Gestire dataset e mitigare bias nei sistemi AI
- Stabilire meccanismi di trasparenza e registri AI
- Applicare human oversight e valutazioni d'impatto (DPIA/IAIA)
- Raggiungere 70% nel quiz sommativo su governance AI

**Argomenti/Topics:**

**3.1 AI Policy aziendale**
- Cosa deve contenere una AI policy conforme:
  - Principi etici e valori aziendali
  - Ruoli e responsabilità (AI owner, compliance officer, DPO)
  - Processo approvazione nuovi sistemi AI
  - Gestione rischi e incidenti
  - Formazione e awareness
- Template policy (1-2 pagine) scaricabile
- Chi approva la policy (board, CEO, compliance)
- Frequenza revisione (annuale o ad-hoc)

**3.2 Dataset quality e mitigazione bias**
- Requisiti AI Act su dataset (qualità, rappresentatività, governance)
- Come identificare bias:
  - Bias nei dati storici (es. recruitment)
  - Bias di selezione (campioni non rappresentativi)
  - Bias di misurazione (proxy variables problematiche)
- Tecniche pratiche di mitigazione:
  - Audit dataset pre-training
  - Diversificazione fonti dati
  - Test su sottogruppi (genere, età, nazionalità)
  - Monitoraggio continuo output
- Checklist: "Il tuo dataset è AI Act-compliant?"

**3.3 Trasparenza e informazione utenti**
- Obbligo trasparenza per sistemi a rischio limitato:
  - Informare utenti che interagiscono con AI
  - Watermark per contenuti AI-generated (immagini, testi, video)
  - Linguaggio chiaro e accessibile
- Obbligo trasparenza per sistemi ad alto rischio:
  - Spiegabilità decisioni automatizzate
  - Diritto a contestazione e intervento umano
  - Informativa privacy specifica
- Esempi pratici: disclaimer chatbot, notice su email AI-generated, spiegazione decisione HR

**3.4 Registri e documentazione AI**
- Registro sistemi AI: cosa tracciare
  - Nome sistema e fornitore
  - Categoria rischio AI Act
  - Funzione aziendale e responsabile
  - Dataset utilizzati
  - Log modifiche e incident
- Documentazione tecnica per sistemi ad alto rischio:
  - Descrizione funzionamento (anche black box: input/output)
  - Valutazione performance e accuracy
  - Misure sicurezza e robustezza
- Template registro AI (Excel/Google Sheet) scaricabile

**3.5 Human oversight (supervisione umana)**
- Cosa significa human-in-the-loop, human-on-the-loop, human-in-command
- Quando è obbligatorio (sistemi ad alto rischio)
- Come implementarlo:
  - Decisioni critiche sempre revisionate da umano
  - Possibilità override decisione AI
  - Alert per decisioni anomale
  - Training operatori su limiti AI
- Esempi: HR manager che valida shortlist AI, credit officer che approva scoring AI

**3.6 Valutazioni d'impatto: DPIA e IAIA**
- DPIA (Data Protection Impact Assessment) per GDPR + AI:
  - Quando obbligatoria (trattamento dati ad alto rischio)
  - Come integrarla con AI Act
  - Template DPIA con sezione AI-specific
- IAIA (Impact Assessment IA) per AI Act:
  - Obbligatoria per sistemi ad alto rischio
  - Valuta impatto su diritti fondamentali
  - Misure mitigazione rischi
- Chi conduce valutazioni (DPO, legal, compliance, consulenti)
- Frequenza: prima del deployment, poi revisione annuale o ad-hoc

**Attività di apprendimento:**

**Lezione interattiva con domande in-line (18-20 min):**
- Ogni sezione (3.1-3.6) ha:
  - Contenuto testuale (150-200 parole)
  - 1-2 domande in-line per verifica comprensione
  - Esempio pratico scaricabile (template, checklist)
- Domande in-line tipo:
  - "Chi deve approvare la AI policy nella tua azienda?" [Open text o choice]
  - "Il tuo dataset di recruitment include solo CV degli ultimi 5 anni, tutti da LinkedIn. Quale bias potrebbe esserci?" [Choice con feedback]
  - "Vero/Falso: Un chatbot deve sempre informare l'utente che è un AI" [V/F]
  - "Quale ruolo ha human oversight in un sistema di credit scoring?" [Scenario]

**Quiz sommativo (10 domande, 12-15 min):**
- Soglia superamento: 70% (7/10 risposte corrette)
- Massimo 2 tentativi
- Mix tipologie:
  - 4 multiple choice (1 risposta corretta)
  - 3 multiple response (2+ risposte corrette)
  - 2 vero/falso con giustificazione
  - 1 scenario-based (caso pratico con 3 sotto-domande)
- Domande coprono tutti i 6 topic del modulo
- Feedback finale con recap errori e link a sezioni da rivedere

**Materiali scaricabili integrati:**
- Template AI Policy (Word/PDF)
- Checklist Dataset Quality (PDF)
- Template Registro AI (Excel/Google Sheet)
- Checklist DPIA/IAIA (PDF)

**Tipo di valutazione:** Sommativa (quiz con soglia 70%; traccia punteggio in SCORM `cmi.core.score.raw`)

**Note per il content creator:**
- **Tono:** Operativo e pratico, "ecco cosa fare lunedì mattina"
- **Esempi:** Template già compilati con esempi fittizi (azienda manifatturiera, retail, servizi)
- **Interattività:** Domande in-line frequenti (ogni 2-3 paragrafi) per mantenere attenzione
- **Linguaggio:** Spiega acronimi (DPIA, IAIA, DPO) alla prima menzione; glossario sempre accessibile
- **Visuals:** Checklist visive, flowchart per human oversight, tabella comparativa DPIA vs IAIA
- **Quiz sommativo:** Domande pratiche, no teoria pura ("Quale azione è conforme..." vs "Quale articolo dice...")
- **Scaricabili:** File professionali, personalizzabili, con istruzioni brevi (1 paragrafo) su come usarli

### Modulo 4: Piano 90 giorni

**Durata stimata:** 25-30 minuti

**Obiettivi specifici del modulo:**
- Pianificare 5 azioni concrete per la conformità AI Act nei prossimi 90 giorni
- Assegnare ruoli e responsabilità per ciascuna azione
- Valutare fornitori AI secondo criteri AI Act
- Organizzare comunicazione interna e formazione
- Produrre un mini-piano operativo di 1 pagina pronto all'uso

**Argomenti/Topics:**

**4.1 Roadmap 90 giorni: le 5 azioni chiave**
- **Azione 1 (Settimana 1-2):** Inventory AI - Censimento sistemi AI in uso
  - Coinvolgere IT, marketing, HR, operations
  - Usare template registro AI (Modulo 3)
  - Output: lista completa sistemi AI con classificazione rischio
- **Azione 2 (Settimana 2-4):** AI Policy - Redigere e approvare policy aziendale
  - Coinvolgere legal, compliance, CEO/board
  - Personalizzare template policy (Modulo 3)
  - Output: AI policy approvata e comunicata
- **Azione 3 (Settimana 3-6):** Gap analysis - Valutare gap di conformità
  - Per sistemi ad alto rischio: DPIA/IAIA
  - Per tutti i sistemi: trasparenza, dataset quality, human oversight
  - Output: lista gap e priorità interventi
- **Azione 4 (Settimana 4-8):** Vendor due diligence - Valutare fornitori AI
  - Richiedere documentazione conformità AI Act
  - Clausole contrattuali AI Act-compliant
  - Output: questionario fornitori, clausole standard
- **Azione 5 (Settimana 6-12):** Formazione e comunicazione - Awareness interna
  - Formazione ruoli chiave (IT, HR, marketing, legal)
  - Comunicazione dipendenti su AI policy
  - Output: piano formazione, materiali comunicazione

**4.2 Matrice ruoli e responsabilità**
- Chi fa cosa:
  - **CEO/Board:** Approvazione policy, budget, sponsorship
  - **Legal/Compliance:** Redazione policy, DPIA/IAIA, contratti fornitori
  - **IT/Data:** Inventory sistemi, dataset quality, registro AI
  - **HR:** Formazione, comunicazione, compliance recruitment AI
  - **Marketing:** Trasparenza contenuti AI-generated, chatbot
  - **DPO:** DPIA, coordinamento GDPR+AI Act
- Template RACI matrix scaricabile (Excel)

**4.3 Vendor AI due diligence**
- Cosa chiedere ai fornitori AI (OpenAI, Microsoft, Google, vendor SaaS):
  - Documentazione conformità AI Act
  - Categoria rischio del sistema fornito
  - Misure sicurezza e robustezza
  - Gestione dataset e bias
  - Procedure incident e supporto
- Questionario vendor AI (10-15 domande) scaricabile
- Clausole contrattuali AI Act:
  - Obbligo conformità normativa
  - Diritto audit e ispezione
  - Responsabilità in caso non conformità
  - Supporto per DPIA/IAIA
- Checklist negoziazione contratti AI

**4.4 Comunicazione interna e change management**
- Strategia comunicazione:
  - Announcement AI policy (email CEO, intranet)
  - FAQ dipendenti su AI Act
  - Esempi pratici per ciascuna funzione
  - Tone: opportunità innovazione responsabile, non "nuovi obblighi burocratici"
- Piano formazione:
  - **Awareness generale** (30 min, tutti i dipendenti): cos'è AI Act, AI policy aziendale
  - **Formazione ruoli chiave** (60-90 min, IT/HR/marketing/legal): approfondimento specifico funzione
  - **Workshop pratico** (mezza giornata, compliance team): DPIA, registro, vendor due diligence
- Template comunicazione scaricabili (email, slide deck, FAQ)

**4.5 Metriche e monitoraggio**
- KPI conformità AI Act:
  - % sistemi AI censiti e classificati (target 100% entro 90 giorni)
  - AI policy approvata (si/no, deadline)
  - DPIA completate per sistemi ad alto rischio (numero)
  - Fornitori AI valutati (% su totale fornitori AI)
  - Dipendenti formati (% su popolazione target)
- Dashboard monitoraggio (template Excel/Google Sheet)
- Frequency review: mensile primi 6 mesi, poi trimestrale

**Attività di apprendimento:**

**Activity builder interattivo (12-15 min):**
- Partecipante costruisce il proprio piano 90 giorni selezionando:
  - **Azioni prioritarie** (sceglie 5 tra 8-10 proposte, con suggerimenti basati su dimensione azienda e settore)
  - **Ruoli responsabili** (assegna owner per ciascuna azione da dropdown)
  - **Timeline** (drag & drop azioni su calendario 12 settimane)
  - **Risorse** (stima ore/persona, budget se applicabile)
- Output: piano 90 giorni visualizzato in Gantt semplificato
- Possibilità download piano come PDF o Excel

**Compito opzionale - Mini-piano 1 pagina (10-12 min):**
- Partecipante compila template mini-piano operativo:
  - Sezione 1: Contesto azienda (settore, dimensione, sistemi AI principali)
  - Sezione 2: 5 azioni con owner, deadline, deliverable
  - Sezione 3: Rischi e mitigazioni
  - Sezione 4: KPI e milestones
- Template strutturato (Word/PDF compilabile)
- Rubrica valutazione (se corso B2B con tutorship):
  - **Completezza** (25%): Tutte le sezioni compilate
  - **Fattibilità** (25%): Timeline e risorse realistiche
  - **Allineamento AI Act** (25%): Azioni coprono requisiti chiave
  - **Chiarezza** (25%): Deliverable e responsabilità chiari
- Upload opzionale per feedback (se previsto supporto tutorship)

**Tipo di valutazione:** Formativa applicata (activity builder traccia completamento; mini-piano è opzionale, valutato solo se con tutorship)

**Note per il content creator:**
- **Tono:** Motivante e action-oriented, "Da domani puoi partire con..."
- **Esempi:** Piani già compilati per 3 profili tipo (PMI manifatturiera 50 dip., PMI servizi 20 dip., PMI tech 100 dip.)
- **Interattività:** Activity builder deve essere intuitivo (drag & drop, visual calendar, preview piano in tempo reale)
- **Linguaggio:** Concreto, evita "dovresti" preferisci "ti consigliamo" o "azione prioritaria"
- **Visuals:** Gantt semplificato colorato, RACI matrix chiara, icone per ruoli
- **Scaricabili:** Tutti i template professionali e brandizzabili (azienda può inserire logo)
  - Template piano 90 giorni (Excel con formule)
  - RACI matrix (Excel)
  - Questionario vendor (Word/PDF)
  - Clausole contrattuali (Word)
  - Template comunicazione (Word: email, slide deck PPT, FAQ doc)
  - Dashboard KPI (Excel con grafici)
- **Mini-piano:** Template con istruzioni inline ("Inserisci qui...", "Max 50 parole", etc.)

- **Personaggi ricorrenti (opzionale):** 3 PMI fittizie seguite lungo tutto il corso per continuità narrativa

**Linguaggio inclusivo e accessibile:**
- **Accessibilità cognitiva:**
  - Frasi brevi (max 20-25 parole)
  - 1 concetto per paragrafo
  - Liste bullet per info multiple
  - Grassetto per keyword, NO CAPS
- **Terminologia:**
  - Spiega acronimi alla prima menzione
  - Glossario popup sempre disponibile
  - Evita doppi tecnicismi ("L'IAIA valuta l'impatto sui diritti fondamentali" NO → "La valutazione d'impatto IA analizza..." SI)

- **Pagine di contenuto:** 150-200 parole max (2-3 min lettura)

- **Attività interattive:** Frequenti (ogni 2-3 pagine di contenuto)

- **Domande in-line:** Non decorative, devono verificare comprensione o personalizzare percorso
- **Feedback formativo:** Mai solo "Corretto/Sbagliato", sempre spiegazione breve (1-2 frasi)

### SCORM API - Funzioni e data model

**Inizializzazione:**
```javascript
// scormfunctions.js
var API = null;

function findAPI(win) {
  // Standard SCORM API discovery
  while (win.API == null && win.parent != null && win.parent != win) {
    win = win.parent;
  }
  return win.API;
}

function initSCORM() {
  API = findAPI(window);
  if (API) {
    API.LMSInitialize("");
    // Recupera stato precedente
    var lessonStatus = API.LMSGetValue("cmi.core.lesson_status");
    var suspendData = API.LMSGetValue("cmi.suspend_data");
    // Resume logica corso
  }
}
```

**Elementi data model utilizzati:**

| **Elemento CMI** | **Uso nel corso** | **Valori** |
|------------------|-------------------|------------|
| `cmi.core.lesson_status` | Traccia completamento corso | "incomplete", "completed", "passed", "failed" |
| `cmi.core.score.raw` | Punteggio quiz Modulo 3 | 0-100 (numero) |
| `cmi.core.score.min` | Score minimo | 0 |
| `cmi.core.score.max` | Score massimo | 100 |
| `cmi.suspend_data` | Salva progresso (modulo corrente, risposte quiz, activity builder) | String <4096 char |
| `cmi.core.session_time` | Tempo sessione corrente | Format HHMM:SS.SS |
| `cmi.core.total_time` | Tempo totale cumulativo | Format HHMM:SS.SS |
| `cmi.core.exit` | Tipo uscita | "suspend", "logout", "" (completato) |

**Logica completamento:**

```javascript
// Corso completato quando:
// 1. Tutti i 4 moduli visitati
// 2. Quiz Modulo 3 superato (score >= 70)
// 3. Activity builder Modulo 4 completato

function checkCompletion() {
  var mod1Complete = localStorage.getItem('mod1_complete') === 'true';
  var mod2Complete = localStorage.getItem('mod2_complete') === 'true';
  var mod3QuizScore = parseInt(localStorage.getItem('mod3_quiz_score') || 0);
  var mod4PlanComplete = localStorage.getItem('mod4_plan_complete') === 'true';

  if (mod1Complete && mod2Complete && mod3QuizScore >= 70 && mod4PlanComplete) {
    API.LMSSetValue("cmi.core.lesson_status", "passed");
    API.LMSSetValue("cmi.core.score.raw", mod3QuizScore.toString());
  } else if (mod1Complete || mod2Complete || mod3QuizScore > 0 || mod4PlanComplete) {
    API.LMSSetValue("cmi.core.lesson_status", "incomplete");
  }

  API.LMSCommit("");
}
```

### Quiz - Implementazione tecnica

**Quiz formativo Modulo 1 (6 domande):**
- Feedback immediato dopo ogni risposta
- Tentativi illimitati
- Non influenza `cmi.core.lesson_status`
- Traccia solo tentativi in `suspend_data` per analytics

**Quiz sommativo Modulo 3 (10 domande):**
- Massimo 2 tentativi
- Soglia: 70% (7/10 risposte corrette)
- Setta `cmi.core.score.raw` e `cmi.core.lesson_status`
- Al secondo fallimento: messaggio "Rivedi contenuti Modulo 3" + link a sezioni con errori
- Randomizzazione domande (da pool 20, mostra 10) solo al retry

**JavaScript quiz logic:**

```javascript
function submitQuiz() {
  var totalQuestions = 10;
  var correctAnswers = 0;

  for (var i = 1; i <= totalQuestions; i++) {
    var questionId = 'q3_0' + i;
    var selected = document.querySelector('input[name="' + questionId + '"]:checked');

    if (selected && selected.dataset.correct === 'true') {
      correctAnswers++;
    }
  }

  var score = Math.round((correctAnswers / totalQuestions) * 100);

  API.LMSSetValue("cmi.core.score.raw", score.toString());
  API.LMSSetValue("cmi.core.score.min", "0");
  API.LMSSetValue("cmi.core.score.max", "100");

  if (score >= 70) {
    API.LMSSetValue("cmi.core.lesson_status", "passed");
    showResults(score, true);
  } else {
    var attempts = parseInt(localStorage.getItem('mod3_quiz_attempts') || 0) + 1;
    localStorage.setItem('mod3_quiz_attempts', attempts.toString());

    if (attempts >= 2) {
      API.LMSSetValue("cmi.core.lesson_status", "failed");
      showResults(score, false, true); // true = max attempts reached
    } else {
      showResults(score, false, false);
    }
  }

  API.LMSCommit("");
}
```
